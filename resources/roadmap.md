# Roadmap · 4–6 周动手计划

> 原则：每周一个可动手任务 + 一个公开产出。卡住时才回查吴恩达课程，别当主线。

| 周 | 动手主线 | 吴恩达查漏（按需） | 公开产出（中英双语） |
|---|---|---|---|
| **W1** | 跑通 + 改 `linreg.py` 三个实验；玩透梯度下降可视化 | C1《监督学习》梯度下降节 | 「我用 30 行手写了梯度下降」+ 可视化录屏 |
| **W2** | 换 PyTorch 重写线性回归，体会 `loss.backward()` | C1 多元回归 | 「backward() 到底替我做了什么」对比手写版 |
| **W3** | Kaggle **Titanic**，跑出第一个真实提交分数 | C2 分类/逻辑回归 | GitHub repo + notebook |
| **W4** | Kaggle **House Prices**（接「面积→房价」锚点） | C2 正则化、过拟合 | 「房价预测里我踩的 3 个坑」 |
| **W5** | 加一层非线性，手搭两层神经网络拟合曲线 | C2 神经网络 | 可视化第二弹：复合函数 = 多层网络 |
| **W6** | Kaggle **Digit Recognizer**（进入 CV）；或起 nanoGPT | C2 多分类 / DL 系列开篇 | 系列收尾：「从函数到神经网络，6 周」 |

## 两条铁律

1. **每周必须有 GitHub commit 或一篇内容产出**——没产出就是又掉回纯听课。
2. 吴恩达的课**只在卡住时回去查**，证书顺手拿（申美硕有用），但绝不当主线。

## 为什么是这个顺序

- 已经手写过梯度下降（W1），所以 W2 用 PyTorch 时能真正体会框架替你做了什么，而非黑箱
- Titanic（分类）→ House Prices（回归）→ Digit Recognizer（深度学习/CV），难度和概念逐级递增
- Transformer 放最后：等前面骨架全认识了，nanoGPT 才看得顺，否则会被 attention 等专属机制带偏学习目标
