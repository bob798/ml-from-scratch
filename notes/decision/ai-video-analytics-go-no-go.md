# 上帝视角：AI 视频分析这事该不该做？

> 不站在"卖方案"立场，而是站在 **投资人 + 产品总监 + 资深架构师** 的复合视角，
> 客观判断这件事的真相、要用什么方法论、关注什么、怎么决策。
>
> 关联：场景方案见 [../industry/ai-video-analytics.md](../industry/ai-video-analytics.md)。

---

## 核心结论（先看这一行）

> **这个市场已经不是"谁的算法准谁赢"，而是"谁能把场景理解 + 服务交付 + 持续运营做到位谁赢"。**
> **如果没有渠道、数据、场景理解、服务能力中至少一项独特优势，就不要下场。**

---

## 一、赛道真相（市场客观判断）

| 维度 | 客观事实 |
|------|---------|
| 市场成熟度 | **红海**，不是蓝海。2017–2020 是红利期，2024+ 进入存量市场 |
| 玩家格局 | 海康/大华（终端+算法+渠道一体）、华为/商汤/旷视（平台+算法）、平安智慧城市/腾讯/阿里（云+生态）、几百家中小集成商和算法公司 |
| 算法壁垒 | **极低**。YOLO/RT-DETR 开源即用，垃圾/入侵/遗留物检测的开源数据集和预训练模型公开可得 |
| 真正的壁垒 | 数据闭环、场景理解、渠道（甲方关系）、服务交付能力、合规资质 |
| 利润分布 | 硬件薄（毛利 15-30%）、算法白菜价、**服务和运维厚（毛利 40-60%）**、数据资产最值钱但难变现 |

## 二、决策框架：5 个杀手级问题

进入前先问自己，每一个答不上就该慎重：

```
Q1. 我有什么是别人没有的？
    - 渠道（拿单能力、甲方关系）
    - 场景数据（某行业的独家数据）
    - 资质（安防/政府/智慧城市的特殊牌照）
    - 成本（自研芯片、自有产线）
    - 服务能力（运维网络、本地团队）
    都没有？-> 别做，或只做单点项目当陪跑

Q2. 我的客户是谁？是真买单的人吗？
    - To G 政府：账期长、合规重、关系导向
    - To B 大企业：决策链长、要 PoC、要定制
    - To 物业/SMB：付费意愿低、流失率高
    选错客户 = 方法论再对也死

Q3. 我是做产品还是做项目？
    - 产品：标准化、可复制、毛利高、但难做
    - 项目：好拿单、现金流好、但不 scalable，做大就死
    最忌讳：用产品的预期做项目的活

Q4. 客户的"误报率 <= 5%"是真需求还是想象？
    如果客户没经历过 AI 项目 -> 根本不知道 5% 意味着什么
    误报率的定义权 = 项目验收的生死线
    不能把定义权交给客户

Q5. 这事 3 年后还存在吗？
    通用 VLM（Qwen-VL, GPT-4V）在快速吞噬专项检测
    3 年后小模型 + 边缘也许会被云端通用模型替代
    现在做的护城河，3 年后还在不在？
```

## 三、方法论体系（五层）

### Layer 1 — 战略层：选位置（Wardley Map）

按"组件演化阶段"思考：

```
[Genesis 起源] -> [Custom 定制] -> [Product 产品] -> [Commodity 商品]

视频接入协议       已 Commodity（不要在这里花精力）
通用目标检测       已 Commodity（开源模型够用）
场景化算法         Product（有空间，但卷）
行业 SOP 沉淀     Custom（这里才是壁垒）
特定客户运营      Custom（数据飞轮起点）
```

**结论**：不要在 Commodity 层卷价格，要在 Custom 层做积累，再标准化往下推。

### Layer 2 — 业务层：JTBD（Jobs To Be Done）

不要问客户"你要不要 AI 检测"，要问：

> **"你最痛的那次事故是什么？事故发生后你最希望第一时间知道什么？"**

真实需求往往不是"检测垃圾"，而是：
- 物业经理："我不想被业主投诉到老板那里" -> 实际买的是"投诉前预警"
- 城管局："我不想被市领导随机抽查抓现行" -> 实际买的是"领导来之前的清零"
- 商场总监："我不想出大事故上新闻" -> 实际买的是"风险兜底"

**JTBD 决定了产品形态**：是"检测系统"还是"事件管理平台"？是"工具"还是"保险"？

### Layer 3 — 产品层：Lean + Stage Gate

```
Stage 0 - 假设验证（1-2 周，不写代码）
  - 找 3 个真实客户访谈，验证 JTBD 和付费意愿
  - 拿现成开源模型 + 客户视频跑 1 天 demo

Stage 1 - POC（4-6 周，单一场景单一客户）
  - 关卡：客户愿意付 POC 费用吗？不愿意 = 伪需求
  - 关卡：能产出 >= 90% 召回的演示视频吗？

Stage 2 - MVP（3 个月，3-5 个客户）
  - 关卡：第二个客户买的时候，复用率 >= 60% 吗？
  - 关卡：单客户毛利 > 0 吗（算上人力运维）？

Stage 3 - Scale（6-12 个月）
  - 关卡：每新增 10 个客户，团队增加 < 5 人吗？
  - 关卡：续费率 > 80% 吗？
```

**每个 Gate 不通过就停**，不要因为"已经投入了"硬着头皮往下走（沉没成本陷阱）。

### Layer 4 — 工程层：MLOps + 渐进交付

```
数据闭环（最关键）：
  采集 -> 标注 -> 训练 -> 部署 -> 反馈 -> 再采集
        |                              |
        +----------- 自动化飞轮 --------+

没有数据闭环的 AI 项目 = 一次性买卖 = 没有壁垒
```

**关键工程原则**：
- **不追求一次到位**：先 80 分上线，靠数据迭代到 95 分
- **离线评估 + 灰度发布**：每次模型更新都要可回滚
- **可解释性 > 准确率**：能告诉客户"为什么误报"比少 1% 误报率重要
- **人在回路（Human-in-the-loop）**：把人工审核做成产品的一部分，而不是 bug

### Layer 5 — 商业层：单位经济模型

```
单客户年度收入 = 硬件分摊 + 软件订阅 + 服务费
单客户年度成本 = COGS（硬件 BOM）+ 实施人力 + 运维人力 + 算力 + 客户成功

LTV / CAC 比 > 3 才健康
回本周期 < 18 个月才安全
```

如果一个客户的实施需要 2 个工程师跑 1 个月，单子又只有 20 万 —— 这不是 AI 公司，是劳务派遣。

## 四、最容易踩的 7 个陷阱

| # | 陷阱 | 真相 |
|---|------|------|
| 1 | "我们算法精度比竞品高" | 客户分不出 92% 和 94% 的区别，但能感知服务响应快不快 |
| 2 | "客户说要什么我们就做什么" | 客户提需求时是想象的，不是真实付费动机 |
| 3 | "先把平台做大再说" | 没有 1 个赚钱的场景之前做平台 = 烧钱 |
| 4 | "误报率 5% 写进合同" | 没有共同的"误报"定义，客户可以让你永远验收不了 |
| 5 | "边做项目边攒产品" | 项目方向太散，最后产品里全是客户 A、B、C 的开关 |
| 6 | "AI 团队 + 销售团队 = 业务" | 缺了交付/客户成功团队，AI 公司 90% 在这里死 |
| 7 | "等通用 VLM 成熟了我们就用" | 通用 VLM 成熟那天，你的算法壁垒立刻归零，要提前规划转型路径 |

## 五、决策矩阵：用什么姿势进入？

| 你的身份 | 推荐姿势 | 为什么 |
|---------|---------|--------|
| 有甲方关系的集成商 | 拿单为主，算法用合作伙伴的（海康/商汤 OEM）| 你的优势是渠道，不是技术 |
| 有行业数据的垂直公司（如物业 SaaS 已有客户）| 嵌入式做增值模块，不单独立项 | 复用现有客户，边际成本低 |
| 算法/AI 创业公司 | 选一个超细分场景做透（如"医院走廊跌倒检测"）| 通用赛道打不过大厂，垂直做透才有定价权 |
| 大厂/巨头 | 平台化 + 生态，开放算法市场 | 用规模碾压，自己不做细分场景 |
| 没渠道没数据没行业 | **不要做这件事** | 没有独特优势就是给红海赛道送钱 |

## 六、下场前的最后三连问

1. **如果把算法部分外包给合作伙伴（用海康/商汤的 SDK），我的业务还成立吗？**
   - 不成立 -> 你没有真正的业务模式，只是在卷算法
   - 成立 -> 恭喜，你找到了真正的差异化

2. **第 10 个客户的交付成本，是不是低于第 1 个客户的 30%？**
   - 不是 -> 你做的是项目，不是产品，规模化即亏损
   - 是 -> 有 scale 潜力

3. **3 年后通用 VLM 把所有专项检测做完了，我的客户为什么还要付钱给我？**
   - 答不上来 -> 现在的护城河是暂时的，要规划第二曲线
   - 答得上来（数据、场景、服务、合规等）-> 这就是真正的壁垒

## 总结一句话

> "AI 视频分析"这件事，技术早就不是问题。
> 问题是你能不能把**"识别一个事件"变成"解决一个业务流程"**。
> 前者是 demo，后者是生意。
> 能完成这个跨越的公司不到 10%，其余的都是陪跑。
