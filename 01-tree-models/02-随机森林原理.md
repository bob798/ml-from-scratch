# 02 · 随机森林原理（Random Forest）

> 配套可视化:viz/random-forest.html(看一棵树找最佳切分 + N棵树投票)
> 英文锚定:decision tree / tree ensemble / bagging / sampling with replacement / out-of-bag

## 从一棵树到一片森林

**单棵决策树(decision tree)** 的核心动作:在每个节点找"最能分开类别的那一刀"
(split),把数据切成子集,再在子集上继续切,直到纯净。

**单棵树的致命弱点:对数据微小变化高度敏感(sensitive to small changes)。**
换掉一两个训练样本,根节点选的特征都可能变,整棵树大变样 —— 这叫高方差(high variance)。
后果:在没见过的新数据上不稳、不可靠。

> 注意:弱点不是"学不够",而是"太敏感/太容易过拟合"。不受限的树能把每个样本
> 切进单独叶子,训练集100%全对,却把噪声也背了下来。

## 解法:树集成(tree ensemble)

不建一棵,建很多棵,让它们**投票**。一棵树敏感,但一群树投票就稳(robust):
个别树因数据微变而出的错,被多数票抹平。

随机森林 = 树集成的一种最常用实现。

## 关键问题:怎么造出很多"不同"的树?

只有一份训练数据,若每棵树都用完整同一份 → 长得一模一样 → 投票=1棵树,无意义。
所以要给每棵树喂"略有不同"的数据。两层随机:

### 第一层:有放回抽样(sampling with replacement / bagging)
从原始数据有放回地抽样(抽完放回,所以会重复+缺席),造出一份 bootstrap 样本。

```
原始 [1,2,3,4,5]
有放回抽5个 → [3,1,3,5,3]   (3出现3次, 2和4缺席)
```

每棵树用一份不同的 bootstrap 样本训练 → 树各不相同、各错不同。
平均约37%的样本不会被抽中,叫袋外样本(out-of-bag, OOB),可当现成测试集。

### 第二层:特征随机(feature randomness)
每次分裂时,只随机看一部分特征里选最佳切分 → 树更加多样,不会都挤着用同一个强特征。

## 为什么投票有效:错误抵消

每棵树过拟合到**不同**的噪声上(张三背错A处,李四背错B处)。
投票时,各自的噪声错误对不上、互相抵消,只有大家都同意的**规律**留下来。

> 这就是"规模化的价值":不在"树多",在"树多样且独立"。
> 100棵完全相同的树投票毫无意义 —— 价值来自独立性。

## 能力边界(很重要)
- 森林投票**对治**:单棵树的敏感/高方差/过拟合 ✅
- 森林投票**治不了**:数据分布不均、数据覆盖缺口 ❌
  (每棵树看的还是同一个偏的分布,投票投不出它没见过的类别)

## 一句话链条
```
单棵树敏感(数据微变→树大变, high variance)
  → 建很多树投票(ensemble)变稳健(robust)
  → 用有放回抽样+特征随机造出"不同"的树
  → 树各错不同 → 投票抵消噪声 → 留下规律 → 防过拟合、泛化好
```

## sklearn 关键参数(调它们=调过拟合)
| 参数 | 调什么 | 对应概念 |
|---|---|---|
| n_estimators | 树的数量 | 投票防过拟合,越多越稳 |
| max_depth | 树多深 | 深=复杂=易过拟合;浅=可能欠拟合 |
| min_samples_leaf | 叶子最小样本数 | 越大越简单,防过拟合 |
| max_features | 每次分裂看几个特征 | 特征随机那一层,制造多样性 |
