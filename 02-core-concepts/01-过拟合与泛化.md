# 07 · 过拟合与泛化(学规律还是背答案)

> 配套可视化:code/viz/overfitting.html(拖复杂度滑块,看欠拟合→刚好→过拟合)
> 这是 ML 的终极目标,也是后面调参/特征/评价指标的地基。

## 一句话

**过拟合 = 模型把训练数据里的偶然噪声也当成规律学了进去。**
**泛化 = 模型在没见过的数据上也表现好的能力 —— 这是 ML 的全部意义。**

## 三种状态

| 状态 | 复杂度 | 训练误差 | 测试误差 | 类比 |
|---|---|---|---|---|
| 欠拟合 | 太低 | 大 | 大 | 只会背"万物皆直线",太笨 |
| 刚刚好 | 适中 | 低 | 低 | 理解了原理,新题也会做 ✅ |
| 过拟合 | 太高 | 极低 | 暴涨 | 背下练习册答案,一考新题就崩 |

关键指纹:**复杂度↑,训练误差一路降,测试误差先降后升(U型)**。甜区在 U 型底部。

## 逐层深挖

**第1层 · 是什么**
训练数据 = 真实规律 + 噪声。模型分不清哪是规律哪是噪声,自由度够大时会贪心拟合每个点(含噪声点),学成"这些点的样子"而非"背后规律"。

**第2层 · 为什么复杂度高就过拟合**
复杂度=自由度。低→只能画直线,没能力迁就噪声(可能欠拟合);高→能扭成任意形状,于是真去穿过每个噪声点。双刃剑,中间有甜区。

**第3层 · 训练误差低不能证明模型好(最反直觉)**
训练表现好,可能是"学到规律"也可能是"背下答案",光看训练误差区分不了。
能证明模型好的唯一办法:**用没见过的数据考它**(测试集,训练时绝不可见)。

**第4层 · 泛化是终极目标**
若只需在见过的数据上对,建查找表即可,不需要ML。ML的价值=从有限例子提炼出能用于无限新情况的规律。所以ML真正衡量的是泛化,不是拟合。

**第5层 · 随机森林为什么能防过拟合**
单棵决策树极易过拟合(能把每个样本切进单独叶子=纯背)。森林靠两个随机:
1. 每棵树只看随机一部分数据(行采样)
2. 每棵树只看随机一部分特征(列采样)
→ 每棵树过拟合到不同噪声 → 投票时各自的噪声互相抵消 → 只留下大家都同意的规律 → 泛化变好。
这印证了"投票的价值来自独立性":正因各树过拟合在不同地方,投票才有意义。

## 链条

```
训练数据 = 真实规律 + 噪声
  复杂度太低 → 欠拟合(都差)
  复杂度太高 → 过拟合(训练好,测试崩)
  复杂度刚好 → 泛化好 ✅
光看训练误差会被骗 → 用没见过的数据考(测试集)
终极目标 = 泛化,不是拟合训练数据
随机森林 = 多样的树各自过拟合不同噪声 → 投票抵消 → 防过拟合
```

## 实践含义(接 Kaggle / 真实项目)
- 永远留测试集/用交叉验证,别用训练分数judge模型
- 训练分数远高于测试分数 = 过拟合信号,该降复杂度/加正则/加数据
- 这是后面理解"正则化""dropout""early stopping"的总根
