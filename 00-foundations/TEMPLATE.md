# 章节写作模板（在 01 函数与映射上验证过的结构）

> 目的：每一篇基础笔记都遵循同一条「认知动线」，做到**高效、精炼、有高度、可查证**。
> 用法：复制下面骨架，逐节填空。带 ⭐ 的是必填，缺了就破坏动线。

## 认知动线（从上往下，读者不迷路）

```
主线总览 → 动机 → 依据(2根) → 是什么(定义→反例→映射) → 代码上手 → 核心机制 → 兜底/边界 → 验收 → 术语 → 出处
```

---

## 骨架

```markdown
# 0X · 篇名

> 一句话价值：搞懂这篇能解锁什么。
>
> 📍 系列位置：上一篇 → **本篇** → 下一篇　｜　⏱ 约 N 分钟

## 一句话与主线 ⭐
- **一句话**：把整篇压成一句。
- **主线**：一条箭头链，后面每节都挂在它上面。
- 提前点破后文要用、但还没定义的关键词（修前向引用）。

## 动机：解决什么问题
为什么需要它。配 **ML 锚点**（落到具体 ML 场景）。

## 依据：别背，整篇只有「N 条根 + 推论」 ⭐
- 列出 ≤3 条「根」（定义 + 动机）。
- 用表格把每条结论映射回某条根（让读者推，不让读者背）。
- 给「自检你是不是在背」的方法。

## 是什么：定义 → 反例 → 映射 ⭐
- 最小可用定义（一句）。
- 反例划边界（看清哪些「不是必须」）。
- 直觉模型 / 类比。
- 配 **ML 锚点**。

## 上手：N 行代码摸到它 ⭐
- 最短可运行片段（契合 from-scratch，能 `python` 跑）。
- 代码里的断言/输出要真实正确（提交前实跑一遍）。

## 核心机制
本篇的「命门级」概念，配 **ML 锚点**。

## 兜底 / 边界
反直觉情况、特例、常见坑。

## 验收三问 ⭐
读完能答出 3 个问题 = 过关。

## 术语对照（中英） ⭐
| 中文 | English | 一句话 |

## 出处与延伸（怎么验证不是我编的） ⭐
- 每块结论配教材 / 原始论文，可查证。
- 历史归属有争议的，注明「主流归功」。
```

---

## 配套资产（每篇尽量配齐）

- `code/` — 可运行的最短示例，提交前 `python` 实跑验证输出。
- `viz/` — 自包含 `.html` 交互可视化（直接浏览器打开，无需构建）。
  设计语言见现有文件：深色主题；色板 `--train:#f5b340`(琥珀)、`--true:#4fd6c9`(青)、`--fit:#ff6b5e`(红)；正文 Georgia，标签 ui-monospace；`viewBox` SVG，`max-width:640`。

## 质量红线

1. 需要背的「根」≤ 3 条，其余必须能从根推出来。
2. 代码片段的输出/断言提交前真跑验证。
3. 每个事实性结论可回溯到出处；拿不准的标注争议。
4. 不前向引用未定义的术语（要用先在主线点破）。
