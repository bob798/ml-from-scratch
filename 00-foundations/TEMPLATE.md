# 章节写作模板（在 01 函数与映射上验证过的结构）

> 目的：每一篇基础笔记都遵循同一条「认知动线」，做到**高效、精炼、有高度、可查证**。
> 用法：复制下面骨架，逐节填空。带 ⭐ 的是必填，缺了就破坏动线；带 🔥 的是**深化模块**，目标是把读者从「理解」推到「会用 / 能拆 / 能造」。

## 两层目标

- **第一层 · 看懂（⭐ 部分）**：能复述、能解释。这是基线。
- **第二层 · 真懂（🔥 部分）**：能用到新问题、能划边界、能从头造、能质疑。
  做法浓缩成 5 个动作：**第一性原理动机、真实场景、边界三连、破坏实验、间隔回忆**——它们逼读者动手和发明，而不是点头。

## 认知动线（从上往下，读者不迷路）

```
主线总览 → 动机(不这样会怎样🔥) → 依据(2根) → 是什么(定义→反例→映射+边界三连🔥)
        → 代码上手 → 破坏实验🔥 → 核心机制 → 真实场景拆解🔥 → 兜底/边界 → 验收(+间隔回忆🔥) → 术语 → 出处
```

## 骨架

```markdown
# 0X · 篇名

> 一句话价值：搞懂这篇能解锁什么。
>
> 📍 系列位置：上一篇 → **本篇** → 下一篇　｜　⏱ 约 N 分钟

## 一句话与主线 ⭐
- **一句话**：把整篇压成一句。
- **主线**：一条箭头链，后面每节都挂在它上面。
- 提前点破后文要用、但还没定义的关键词（修前向引用）。

## 动机：解决什么问题（含「不这样会怎样」🔥）
为什么需要它。配 **ML 锚点**。
🔥 **第一性原理动机**：用一句「**不这样会怎样**」点出它要解决的痛——让读者从「发明它的人当时卡在哪」出发，而不是从结论出发。（例：MSE「不平方会怎样」→ 正负抵消，看着没错其实全错。）
🔥 **动机要爬两层，别停在第一层**：
- **浅层**：满足什么条件才算它（仍是「它是什么」的延伸，如「为什么要唯一输出」）。
- **深层（更值钱）**：为什么「有这个概念」本身就值钱——它把什么变成了可操作的对象，**解锁了哪些原来做不到的操作**（复合 / 求导 / 优化 / 统一……）。
- 检验：若你的「动机」换个概念也照样成立，说明还停在浅层；深层一定能落到「这个抽象解锁了 X」。

## 依据：别背，整篇只有「N 条根 + 推论」 ⭐
- 列出 ≤3 条「根」（定义 + 动机）。
- 用表格把每条结论映射回某条根（让读者推，不让读者背）。
- 给「自检你是不是在背」的方法。

## 是什么：定义 → 反例 → 映射 ⭐（含边界三连🔥）
- 最小可用定义（一句）。
- 反例划边界（看清哪些「不是必须」）。
- 直觉模型 / 类比。配 **ML 锚点**。
- 🔥 **边界三连**：给三个挨着边界的例子——「这是 ✓ / 这差一点就不是 / 这看着像其实不是」。理解的深度 = 边界划得有多准。

## 上手：N 行代码摸到它 ⭐
- 最短可运行片段（契合 from-scratch，能 `python` 跑）。
- 代码里的断言/输出要真实正确（提交前实跑一遍）。

## 破坏实验🔥（从「读懂」到「真懂」的关键一步）
- 给 2~3 个「改坏一处」的操作，要求读者**先预测、再跑、对照**。
- 预测错的地方，就是「以为懂其实没懂」的地方。（例：把 lr 调到 1.0，先猜会震荡还是发散？）

## 核心机制
本篇的「命门级」概念，配 **ML 锚点**。

## 真实场景拆解🔥（反向联系生活）
- 不是「举例说明概念」，而是「拿一个真实问题，问这个概念在这里以什么形式出现」。
- 1~2 个生活/工程场景（导航、定价、推荐……），点出概念在其中的化身。

## 兜底 / 边界
反直觉情况、特例、常见坑。

## 验收三问 ⭐（含间隔回忆🔥）
读完能答出 3 个问题 = 过关。
🔥 **间隔回忆**：补一句「三天后合上笔记再答一遍」——隔一段时间的主动回忆，才检验真的留下了。

## 术语对照（中英） ⭐
| 中文 | English | 一句话 |

## 出处与延伸（怎么验证不是我编的） ⭐
- 每块结论配教材 / 原始论文，可查证。
- 历史归属有争议的，注明「主流归功」。
```

## 配套资产（每篇尽量配齐）

- `code/` — 可运行的最短示例，提交前 `python` 实跑验证输出。
- `viz/` — 自包含 `.html` 交互可视化（直接浏览器打开，无需构建）。
  设计语言见现有文件：深色主题；色板 `--train:#f5b340`(琥珀)、`--true:#4fd6c9`(青)、`--fit:#ff6b5e`(红)；正文 Georgia，标签 ui-monospace；`viewBox` SVG，`max-width:640`。

## 质量红线

1. 需要背的「根」≤ 3 条，其余必须能从根推出来。
2. 代码片段的输出/断言提交前真跑验证。
3. 每个事实性结论可回溯到出处；拿不准的标注争议。
4. 不前向引用未定义的术语（要用先在主线点破）。
5. 🔥 深化模块至少配齐「破坏实验 + 边界三连 + 真实场景」三件，光看懂不算完。
