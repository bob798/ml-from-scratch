## 一句话与主线

**一句话**：导数 = 曲线在**某一点**的斜率（陡不陡），靠「把附近两点无限拉近」求出来。

**主线**（后面每节都挂在上面）：

```
斜率(直线处处一样) →曲线每点不同→ 取附近两点逼近 →间距→0(极限)→ 导数 →它本身也是个函数→ ML 用它找最低点
```

## 动机：为什么要会算「陡不陡」

训练模型 = 在损失曲线上找最低点。可你站在曲线上，并不知道谷底在哪——唯一能感知的局部信息就是脚下「**有多陡、朝哪斜**」。斜率就是这个信息；求导就是把它算出来。

**ML 锚点**：梯度下降全靠斜率指路。不会求斜率，模型就是个睁眼瞎，不知道往哪迈步。

## 依据：别背，整篇只有「2 条根 + 推论」

**根 1 · 斜率的定义**：斜率 = 上升量 ÷ 水平移动量 = Δy / Δx（走一步，升多高。Δ 念 delta = 变化量）。

**根 2 · 关键操作**：一个**点**没有「走一步」，没法直接算斜率；那就在它附近取很近的两点算斜率，再让两点间距 → 0，看这个值稳定在哪——稳定值就是这一点的导数（「无限拉近」数学里叫**极限**）。

**从这两条推出来（不用背）：**

| 文中结论 | 从哪条根推出来 |
|---|---|
| 直线处处同斜率 | 根 1：直线上随便取两点，Δy/Δx 都一样 |
| 曲线某点的斜率 | 根 2：取附近两点逼近 |
| x² 的导数是 2x | 根 2 的逼近跑出来的规律，总结成公式，省得每次重算 |
| f'(x) 也是个函数 | 输入「位置 x」→ 输出「该点斜率」，满足唯一对应（挂回 [01](./01-函数与映射.md)）|
| ML 为什么死磕求导 | 根 1 给出方向与陡度 → 决定往哪走、走多急 |

## 是什么：定义 → 直觉 → 公式

**定义**：斜率回答「走一步升多高」。

```
斜率 = Δy / Δx
斜率大 = 陡；小 = 缓；0 = 平地；负 = 下坡
```

直线很简单，处处一样陡，整条线一个斜率：

```
两点 (1,3) 和 (3,7)：斜率 = (7−3)/(3−1) = 2 → x 每走 1，y 升 2
```

**难点**：y = x² 这种曲线，每个地方陡的程度都不一样。一个点没有「走一步」，怎么算？

**放大镜直觉（根 2）**：取它附近两点连线的斜率，当作这一点的斜率；两点越靠越近，值会稳定下来。求 y=x² 在 x=3 处：

```
跨度 1    → 7
跨度 0.1  → 6.1
跨度 0.01 → 6.01
跨度 →0   → 6   ← 这就是 x=3 这一点的真正斜率
```

收敛到的 6，就是导数。**求导（derivative）= 求某一点的斜率。**

## 上手：代码亲眼看它收敛

```python
def f(x): return x**2
x = 3
for h in [1, 0.1, 0.01, 0.001, 0.0001]:
    slope = (f(x + h) - f(x)) / h
    print(f"跨度 {h:<7} → 斜率 {slope}")
# 斜率逐步逼近 6.0
```

## 核心机制：公式从哪来 & 为什么 ML 死磕

**那个公式（x² → 2x）只是偷懒**：数学家发现 x² 在任意点的斜率永远是 2x，于是不必每次都跑逼近过程。

- x=3 → 6 ｜ x=5 → 10 ｜ x=0 → 0（谷底，平的）

**导数 f'(x)=2x 本身又是一个函数**——输入位置，输出该点斜率。

**为什么 ML 死磕**：斜率同时告诉你方向和步长——

```
斜率为正 → 右边在升 → 往左走        斜率大(陡) → 离谷底远 → 迈大步
斜率为负 → 右边在降 → 往右走        斜率小(缓) → 快到了   → 迈小步
斜率 ≈ 0 → 到谷底 → 停
```

这就是**梯度下降**：站在曲线上算斜率，朝下坡迈一步，反复迈到谷底（下一篇展开）。

## 验收三问

1. 斜率是什么？（走一步升多高）
2. 求导在求什么？（曲线上某一点的斜率）
3. 怎么求出来的？（取附近两点算斜率，让间距→0，看稳定值）

## 术语对照（看英文资料时不卡壳）

| 中文 | English | 一句话 |

## 出处与延伸（怎么验证这些不是我编的）

**导数 = 某点斜率 = 差商的极限（2 条根的来源）**
- 任何微积分教材的「导数」定义即 `f'(x)=lim_{h→0}(f(x+h)−f(x))/h`，与本篇放大镜逼近完全一致。
- 教材验证：James Stewart《Calculus》导数章；Rudin《数学分析原理》第 5 章（严格极限定义）。
- 代码里用的是**数值微分（numerical differentiation）**，工程上常用有限差分近似导数。

**为什么 ML 用它**
- Goodfellow、Bengio、Courville《Deep Learning》(2016) 第 4 章（数值计算）、第 8 章（优化）：梯度即多维导数，是优化的核心信息。

> 用法：遇到「为什么是这样」，先回到上面两条根，而不是把 2x 当口诀背。
