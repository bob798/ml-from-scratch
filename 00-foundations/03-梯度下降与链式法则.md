## 一句话与主线

**一句话**：沿斜率「减一步」就能下坡（梯度下降）；复合函数求导 = 各级斜率连乘（链式法则）；反向传播就是它叠很多层。

**主线**（后面每节都挂在上面）：

```
一个减号定方向(梯度下降) → 复合怎么求导(链式法则=连乘) → 叠很多层=反向传播 → 线性回归实战 → 训练三步循环
```

## 动机：怎么把「会算斜率」变成「会自己变好」

上一篇会算斜率了，但斜率只是「脚下信息」。要让模型**自动**爬到谷底，还差两件事：① 一条「根据斜率该怎么迈步」的规则（梯度下降）；② 网络是层层复合的，怎么把末端误差的「责任」算回到每个参数（链式法则）。这两件事凑齐，就是训练。

## 依据：别背，整篇只有「2 条根 + 推论」

**根 1 · 下坡规则**：`x ← x − 学习率 × 斜率`。那个**减号**自动帮你挑对方向，不用记「正左负右」。

**根 2 · 链式法则**：复合函数的总斜率 = **沿途各级斜率连乘**（逐级放大倍数相乘）。

**从这两条推出来（不用背）：**

| 文中结论 | 从哪条根推出来 |
|---|---|
| 「正往左、负往右」不用记 | 根 1：减号代你选方向 |
| 学习率太大震荡 / 太小太慢 | 根 1：步长 = lr × 斜率，lr 直接控步幅 |
| 反向传播 | 根 2 套很多层，从输出端往输入端倒着连乘 |
| loss 对 w 的梯度 = 2·err·x | 根 2：外层 err²→2·err，内层 wx→x，相乘 |
| 反向传播没有玄机 | 根 2 的重复应用，只是层数多 |

## 是什么（一）：梯度下降 = 一个减号

下坡的动作永远是这一行：

```
x ← x − 学习率 × 斜率
```

- 斜率为**正** → 减去正数 → x 变小 → 往**左**
- 斜率为**负** → 减去负数(=加) → x 变大 → 往**右**

**记那个减号就够**——它自动挑对方向。这就是 `w -= lr * grad` 里减号的全部意义。

学习率（lr）= 每步迈多大：太大（如 1.0）在谷底两边横跳甚至跳飞；太小（如 0.001）慢吞吞到不了底。

## 是什么（二）：链式法则 = 复合生产线求导

设 g(x)=2x+1，f(u)=u²，复合 f(g(x))。问：**x 动一点，最终输出 y 动多少？**
难点：x 不直接碰 y，中间隔着 u。链式法则说——**分段算斜率，再乘起来。**

```
x ──g──> u ──f──> y

第一段斜率：u 对 x  → g'(x) = 2
第二段斜率：y 对 u  → f'(u) = 2u = 2×7 = 14
合起来：y 对 x = 2 × 14 = 28
```

**直觉**：x 推动 u 放大 2 倍，u 推动 y 放大 14 倍，总放大 = 2×14 = 28。**链式法则就是「逐级放大倍数连乘」。**

## 上手：代码验证链式法则

```python
def g(x): return 2*x + 1
def f(u): return u**2
x, h = 3, 1e-6
slope = (f(g(x+h)) - f(g(x-h))) / (2*h)
print(slope)   # ≈ 28.0
```

## 核心机制：接到反向传播

神经网络是 f₃(f₂(f₁(x)))，几十上百级生产线。想知道最末端误差对最前面某个参数的影响，就把**沿途每一级斜率全乘起来**。「反向」是因为从输出端往输入端倒着乘回去。

**反向传播 = 复合函数求导。** 没有别的玄机，只是层数多。

## 实战：线性回归里的链式法则

要算 loss 对 w 的斜率。生产线：

```
w ──> 预测 ŷ=wx+b ──> 误差 err=ŷ−y ──> loss=err²
```

- 外层：loss=err²，对 err 求导 = `2*err`
- 内层：err 里只有 wx 跟 w 有关，wx 对 w 求导 = `x`
- 乘起来：loss 对 w 的斜率 = `2*err*x`

对 b 同理：`+b` 对 b 求导 = 1，所以 `2*err*1`。

**直觉**：`2*err` 说「现在错多少、往哪个方向错」；乘上 `x` 说「这个点 x 越大，w 对它影响越大，要更用力调」。

## 完整训练循环（核心三步）

```python
for step in range(200):
    pred = f(x)               # 1. 前向：算预测
    err  = pred - y
    grad_w = np.mean(2*err*x) # 2. 反向：链式法则算梯度
    grad_b = np.mean(2*err*1)
    w -= lr * grad_w          # 3. 下坡：沿斜率走一步
    b -= lr * grad_b
```

跑通后 loss 从 37 降到 0.27，w/b 自己爬到接近真相（2 和 5）。

## 术语对照（看英文资料时不卡壳）

| 中文 | English | 一句话 |
|---|---|---|
| 梯度下降 | gradient descent | 沿斜率反方向迈步找最低点 |
| 学习率 | learning rate | 每步迈多大 |
| 链式法则 | chain rule | 复合求导 = 各级斜率连乘 |
| 反向传播 | backpropagation | 多层复合的链式法则，从后往前 |
| 梯度 | gradient | 多维情形的斜率（每个参数一个）|

## 出处与延伸（怎么验证这些不是我编的）

**链式法则（根 2）**
- 微积分基本结论，任何微积分教材「复合函数求导」一节，如 Stewart《Calculus》。

**梯度下降（根 1）**
- 思想最早可追溯到 Cauchy（1847）的最速下降法；现代深度学习语境见 Goodfellow 等《Deep Learning》(2016) 第 8 章。

**反向传播 = 复合函数求导**
- 经典文献：Rumelhart, Hinton & Williams, *Learning representations by back-propagating errors*, **Nature 323:533–536 (1986)** —— [nature.com/articles/323533a0](https://www.nature.com/articles/323533a0)。
- 教材验证：《Deep Learning》第 6.5 节（反向传播与其他微分算法），明确把 backprop 讲成链式法则的高效实现。
