## 沿斜率下坡：一个减号搞定方向

下坡的动作永远是这一行：

```
x ← x − 学习率 × 斜率
```

- 斜率为**正** → 减去正数 → x **变小** → 往**左**
- 斜率为**负** → 减去负数（=加）→ x **变大** → 往**右**

**不用记「正左负右」，记那个减号就行**——减号自动帮你挑对方向。这就是 `w -= lr * grad` 里减号的全部意义。

学习率（lr）= 每步迈多大：
- 太大（如 1.0）→ 在谷底两边来回横跳甚至跳出去
- 太小（如 0.001）→ 慢吞吞，半天到不了谷底

## 链式法则：复合生产线求导

设 g(x)=2x+1，f(u)=u²，复合 f(g(x))。问：**x 动一点，最终输出 y 动多少？**

难点：x 不直接碰 y，中间隔着 u。链式法则说：**分段算斜率，再乘起来。**

```
x ──g──> u ──f──> y

第一段斜率：u 对 x  → g'(x) = 2
第二段斜率：y 对 u  → f'(u) = 2u = 2×7 = 14
合起来：y 对 x = 2 × 14 = 28
```

**直觉**：x 推动 u 放大 2 倍，u 推动 y 放大 14 倍，总放大 = 2×14 = 28。**链式法则就是「逐级放大倍数连乘」。**

代码验证：

```python
def g(x): return 2*x + 1
def f(u): return u**2
x, h = 3, 1e-6
slope = (f(g(x+h)) - f(g(x-h))) / (2*h)
print(slope)   # ≈ 28.0
```

## 接到反向传播

神经网络是 f₃(f₂(f₁(x)))，几十上百级生产线。想知道最末端误差对最前面某个参数的影响，就把**沿途每一级斜率全乘起来**。「反向」是因为从输出端往输入端倒着乘回去。

**反向传播 = 复合函数求导。** 没有别的玄机，只是层数多。

## 实战：线性回归里的链式法则

要算 loss 对 w 的斜率。生产线：

```
w ──> 预测 ŷ=wx+b ──> 误差 err=ŷ−y ──> loss=err²
```

- 外层：loss=err²，对 err 求导 = `2*err`
- 内层：err 里只有 wx 跟 w 有关，wx 对 w 求导 = `x`
- 乘起来：loss 对 w 的斜率 = `2*err*x`

对 b 同理：`+b` 对 b 求导 = 1，所以 `2*err*1`。

**直觉**：`2*err` 说「现在错多少、往哪个方向错」；乘上 `x` 说「这个点 x 越大，w 对它影响越大，要更用力调」。

## 完整训练循环（核心三步）

```python
for step in range(200):
    pred = f(x)               # 1. 前向：算预测
    err  = pred - y
    grad_w = np.mean(2*err*x) # 2. 反向：链式法则算梯度
    grad_b = np.mean(2*err*1)
    w -= lr * grad_w          # 3. 下坡：沿斜率走一步
    b -= lr * grad_b
```

跑通后 loss 从 37 降到 0.27，w/b 自己爬到接近真相（2 和 5）。
